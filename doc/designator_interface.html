Begin Context

Start a new context with a given symbolic string identifier. Service: /operate Input: Single Designator
Field Value
_name Symbolic name of the context to begin
Output: Single Designator
Field Value
_id The assigned integer identifier for the new context

End Context

End a formerly started context given its integer identifier. This ends all other still open child contexts of this context and marks them as prematurely ended and associates their end time with this context's end time. Service: /operate Input: Single Designator
Field Value
_id The integer identifier of the context to end
Output: No Return Value

Export Files

Export logged data. The type of data to store strongly depends on the plugins loaded. By default, the formats OWL, DOT, and META are accepted. Service: /operate Input: Single Designator
Field Value
command export-planlog
format owl, dot, or meta
filename Filename to use when exporting data to named files. Only valid for OWL, and DOT.
show-successes 0, or 1. Show successful contexts. DOT format only. Default is 1.
show-fails 0, or 1. Show failed contexts. DOT format only. Default is 1.
max-detail-level Integer. Only show contexts that have at maximum the given detail level attribute. Default is 99.
Output: No Return Value

Start New Experiment

Start a new experiment and reinitialize loaded plugins for the new experiment. Service: /operate Input: Single Designator
Field Value
command start-new-experiment
Output: No Return Value

Add Object to Active Context

Associate a given object designator to the currently active context. Service: /operate Input: Single Designator
Field Value
command add-object
type "OBJECT"
memory-address Memory address of the designator to add
description Designator description of the object to add
Output: Single Designator
Field Value
id Semantic identifier (string) of the added object designator.

Set Experiment Metadata

Set a metadata attribute for the active experiment, consisting of a field name and an associated value. Service: /operate Input: Single Designator
Field Value
command set-experiment-meta-data
field String identifier of the field to set
value Value for the field to set
Output: No Return Value

Add Topic Image to Active Context

Trigger capturing of an image from a ROS topic and associate it to the currently active context. Service: /operate Input: Single Designator
Field Value
command add-image
origin ROS topic name to capture the image from
Output: No Return Value

Add Failure to Active Context

Associate a symbolic failure to the active context. The context will automatically be marked as failed. Service: /operate Input: Single Designator
Field Value
command add-failure
condition String identifier of the failure to add
Output: No Return Value

Add Designator to Active Context

Associate a designator to the active context. Service: /operate Input: Single Designator
Field Value
command add-designator
type The type of designator to add. Allowed values are ACTION, OBJECT, and LOCATION.
annotation Meta annotation of the designator to add used by postprocessing plugins.
memory-address Memory address of the designator to add
description Designator description of the designator to add
Output: Single Designator
Field Value
id Semantic identifier (string) of the added object designator.

Equate two Designator

Equate two designators with each other, effectively chaining them for symbolic reasoning. Service: /operate Input: Single Designator
Field Value
command equate-designators
memory-address-child Memory address of the child designator.
type-child Type of the child designator.
description-child Designator description of the child designator.
memory-address-parent Memory address of the parent designator.
type-parent Type of the parent designator.
description-parent Designator description of the parent designator.
Output: No Return Value
